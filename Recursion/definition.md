# 什么是递归
* 将问题分解为规模更小的相同问题
* 持续分解，直到问题规模小到可以用非常简单直接的方式解决
* 递归问题分解方式非常独特，其算法方面的明显特征是：在算法过程中调用自身

## 递归三定律
* 递归算法必须有一个基本结束条件（最小规模问题的直接解决）
* 递归算法必须能改变状态向基本结束条件演进 （减小问题规模）
* 递归算法必须调用自身 （解决减小规模的相同问题）
  
## 求和
```python
#常规方式
def listsum(numList):
    theSum = 0
    for i in numList:
        theSum += i
    return theSum

#递归
def listSum(numList):
    if len(numList) == 1: #结束条件
        return numList[0] #最小规模问题的解决
    else:
        return numList[0] + listSum(numList[1:]) #调用自身，减小相同问题
```

## 递归调用的实现
* 当一个函数被调用的时候，系统会把调用时的现场数据压如到系统调用栈
* 每次调用，压入栈的现场数据称为栈帧
* 当函数返回时，要从调用栈的站定取得返回地址，恢复现场，弹出栈帧，按地址返回
* 在调试递归算法时，当递归层数太多，系统调用栈容量有限：RecursionError。原因检查程序中是否忘记设置基本结束条件，导致无限递归

# 动态规划求解
* 动态规划算法采用一种更有调理的方式来得到问题的解
* 动态规划的前提：问题的最优解包含了更小规模子问题的最优解
